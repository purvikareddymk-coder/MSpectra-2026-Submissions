# MSpectra Hackathon 2026 - Official Submissions

Welcome to the central submission repository for MSpectra 2026! 

## 📌 Submission Instructions for Teams

To avoid merge conflicts, NO TEAM is allowed to push directly to this repository. You must use the Fork & Pull Request workflow.

### Step 1: Fork & Clone (Tonight)
1. The Team Lead must click the **Fork** button at the top right of this repository. This creates a copy in your personal GitHub account.
2. Clone your forked repository to your local machine:
   `git clone https://github.com/YOUR-USERNAME/MSpectra-2026-Submissions.git`

### Step 2: Create Your Team Folder (Tonight)
1. Inside the cloned repository, create a new folder with your exact team name (e.g., `Team_YMCA`).
2. Move all of your project code into this folder. 
3. Inside your team folder, create a `README.md` that includes:
   - **Team Name & Member Names**
   - **Assigned Problem Statement / Company**
   - **Tech Stack Used**

### Step 3: Push & Open a Pull Request (Tonight)
1. Commit and push the code to your forked repository.
2. Come back to this official MISC repository and click **Pull Requests** -> **New Pull Request**.
3. Title the Pull Request with your Team Name (e.g., `Submission: Team YMCA`).
4. Click **Create Pull Request**.

### Step 4: Day 2 Updates (4th April)
Tomorrow, as you finish features and deploy your app, simply commit and push the changes to your local fork. **Your Pull Request will automatically update!** 

By 1:00 PM Tomorrow, your team folder's `README.md` MUST be updated to include:
- 🔗 **Live Deployment Link** (Vercel/Railway/Cloudflare)
- 🎥 **Link to your Video Demo (Public link obtained using Google Drive or OneDrive)**
- 📊 **Link to your Presentation (PPT)**
